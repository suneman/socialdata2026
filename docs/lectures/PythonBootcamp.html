Part 1: Setting Up Your Environment¶

You have several options for running Python notebooks in this course:

Option A: Local Installation with Anaconda (Recommended)¶

For a local setup, we recommend the Anaconda distribution. It includes Python, Jupyter, and most libraries you'll need.

Once installed, you can start notebooks by:

  • Running jupyter notebook in your terminal (classic interface), or
  • Running jupyter lab for the more modern JupyterLab interface (I use Notebooks, but I'm old fashioned)

Option B: Google Colab (Easiest)¶

Google Colab lets you run Jupyter notebooks in your browser with zero installation. It comes with most data science libraries pre-installed. This is a great option if you want to get started immediately or if you're having installation troubles. But then you're sharing your data with Google ... and you might have issues with large files.

Option C: VS Code with Jupyter Extension¶

If you prefer a full IDE, VS Code with the Jupyter extension is excellent. It gives you the notebook experience inside a proper code editor with git integration, debugging, and more.

Whichever you choose¶

There are many other useful setups, so in the end, just make sure you can open this notebook and run the cells below before proceeding.

Part 3: Using LLMs for Coding¶

Large Language Models have transformed how we write code. In this course, we encourage you to use LLMs as coding assistants, but with intention.

When LLMs are helpful¶

  • Boilerplate code: Setting up plots, reading files, standard data transformations
  • Debugging: "Why am I getting this error?" often gets good answers
  • Learning new libraries: "How do I do X in pandas?" is faster than searching docs (although I feel ambivalent about this since it means great sites like stackoverflow are dying)
  • Code explanation: Understanding unfamiliar code you encounter
  • Styling plots and making them look good: If you can tell an LLM what you want a plot to look like, it can save lots of time with matplotlib documentation.

When to be careful¶

  • Core concepts: If you don't understand why something works, you'll struggle to adapt it
  • Analysis decisions: LLMs don't know your data or research questions
  • Blindly copying: Always read and understand code before using it
  • Complex logic: LLMs can confidently produce subtly wrong code

My expectation¶

You should be able to explain any code you submit. If you used an LLM to help write parts of it, that's fine, but make sure you understand what it does and why. The goal is learning, and LLMs are a tool to help you learn. The trouble is that usually it means that you avoid learning.

Subplots¶

Often you'll want multiple plots side by side.

fig, axes = plt.subplots(1, 2, figsize=(12, 4))

# Left plot: population

axes[0].set_ylabel("Population (thousands)")
axes[0].set_title("Population")

# Right plot: density
axes[1].bar(df["city"], df["density"], color="coral")
axes[1].set_ylabel("People per km²")
axes[1].set_title("Population Density")

plt.tight_layout()
plt.show()
